## 1. Auditoría de Backend (NLP Pipeline + API)

### Perspectivas: Lingüista Computacional + Arquitecto Python/FastAPI

#### 1.1 Hallazgos MEDIOS (Requieren Corrección)

**M1. Missing Error Handling en `chapter_summary.py` (Línea 846-857)**
- **Problema**: `sanitize_for_prompt()` no valida resultado vacío antes de `format()` de prompts LLM
- **Riesgo**: Prompts con texto vacío entrenan LLM con inputs inútiles
- **Fix**: Validar `text_to_analyze.strip()` antes de formatear
- **Prioridad**: MEDIA - Corregir antes del próximo sprint

**M2. Race Condition en `_invalidation.py` (Línea 62-123)**
- **Problema**: `emit_invalidation_event()` NO es transaccional completo:
  - Lee `MAX(revision)` → Inserta evento → Marca cachés stale
  - Si falla entre INSERT y UPDATE, cachés quedan desincronizados
- **Riesgo**: Cachés no invalidados correctamente
- **Fix**: Usar SAVEPOINTs SQLite o garantizar que `_mark_stale()` nunca falla (UPSERT)
- **Prioridad**: MEDIA - Importante para coherencia de cache

**M3. Performance Bottleneck: O(N²) en `_analysis_phases.py` (Línea 1931-1992)**
- **Problema**: Comparación de entidades por tipo es O(N²) sin límites
  - 1000 personajes → ~500,000 pares → Cada par llama `fusion_service.should_merge()` (embeddings costoso)
- **Riesgo**: Manuscritos grandes (>500 personajes) tienen tiempos prohibitivos
- **Fix**: Implementar clustering o index de similaridad de nombres antes del O(N²)
- **Prioridad**: MEDIA - Afecta escalabilidad, pero no bloquea uso actual

**M4. Voting Fallback Deficiente en `coreference_resolver.py` (Línea 1617-1632)**
- **Problema**: Sistema de votación no logea cuando `all_votes` queda vacío (todos los métodos fallan)
- **Riesgo**: Errores silenciosos en resolución de correferencias
- **Fix**: Añadir `logger.warning()` cuando no hay votos
- **Prioridad**: BAJA-MEDIA - Solo afecta observabilidad

#### 1.2 Hallazgos BAJOS

**L1. Inconsistencia en Invalidation Semantics** (`_invalidation.py`, línea 22-60)
- `ENTITY_DEPENDENT_TYPES | ATTRIBUTE_DEPENDENT_TYPES` tiene overlap → redundancia
- No es bug funcional, pero ineficiente

**L2. Missing Schema Validation** (`chapter_summary.py`, línea 1089-1104)
- Query asume `entity_mentions` tiene `context_before`/`context_after`
- DB antigua sin esas columnas → falla sin manejo específico

#### 1.3 Puntos Positivos

✅ **Excelente sistema de votación ponderada** (coreference_resolver.py)
- Múltiples métodos con fallbacks graceful
- Logging detallado de contribuciones por método

✅ **Cache invalidation architecture robusta**
- Granular por tipo de enrichment
- Sistema de revisiones para tracking

✅ **Strong sanitization en LLM inputs**
- Uso consistente de `sanitize_for_prompt()`
- Límites de caracteres enforced

✅ **Thread-safe progress storage**
- Uso correcto de `deps._progress_lock`
- Transacciones SQLite con WAL mode

✅ **Cancellation handling**
- Checks periódicos de `tracker.check_cancelled()`
