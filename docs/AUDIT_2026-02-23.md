## 1. Funciones Infrautilizadas — Integración Completada

### 1.1 `are_hypocoristic_match()` — NO usada en correferencia (CORREGIDO)

**Problema**: El diccionario de 130+ hipocorísticos españoles (`semantic_fusion.py`, 1644 LOC) solo se usaba en fusión post-NER y detección de variantes ortográficas. **No participaba** en la resolución de correferencias, que es donde más impacto tiene.

**Impacto**: Cadenas como "Paco dijo... Francisco respondió..." no se enlazaban. "Mari sonrió. María la llamó." generaba dos entidades separadas.

**Solución implementada**:
- `coreference_resolver.py`: Añadido `_check_hypocoristic()` a `MorphoCorefMethod` y `HeuristicsCorefMethod`
  - Bonus de +0.6 cuando anaphor y candidato son PROPER_NOUN con match hipocorístico
  - Lazy import para evitar dependencia circular
- `scope_resolver.py`: Añadido check hipocorístico a `_names_match_flexible()`
  - Permite heredar atributos entre variantes de nombre (Mari hereda atributos de María)

**Tests**: 21 tests nuevos en `test_hypocoristic_coref.py`

### 1.2 `normalize_for_comparison()` — Duplicada 10+ veces (CONSOLIDADO)

**Problema**: La normalización de nombres para comparación estaba reimplementada en 10+ archivos con variaciones sutiles (algunas preservaban ñ, otras no; algunas lowercase, otras no).

**Archivos afectados**: `entity_matcher.py`, `entity_continuity_service.py`, `morpho_utils.py`, `name_variant_detector.py`, `ua_ner.py`, `unified_analysis.py`, `speaker_attribution.py`, `version_diff.py`, `ua_resolution.py`, `semantic_fusion.py`, `manuscript_identity.py`

**Solución**: Ver sección 2 (DRY).

### 1.3 `_token_jaccard()` / `jaccard_similarity()` — Duplicada en 3 archivos (CONSOLIDADO)

**Problema**: Similitud Jaccard implementada localmente en `entity_continuity_service.py`, `entity_matcher.py` y `clustering.py`.

**Solución**: Consolidado en `text_utils.py`.

## 3. Bugs y Defectos Técnicos Corregidos

### 3.1 Bug de Clustering: Contención por Subcadena (CORREGIDO)

**Archivo**: `entities/clustering.py` → `_fast_name_similarity()`

**Problema**: Usaba `n1 in n2` (contención de subcadena) para determinar similaridad. Esto provocaba:
```
"a" in "maria garcia" → True → similaridad 0.77
"el" in "miguel" → True → similaridad 0.82
```
Cualquier nombre corto (1-2 chars) matcheaba con nombres largos.

**Fix**: Cambiado a contención a nivel de **palabras** (word-level set containment):
```python
words1 = set(n1.split())
words2 = set(n2.split())
if words1 and words2 and (words1 <= words2 or words2 <= words1):
    shorter = min(len(n1), len(n2))
    if shorter >= 3:  # Mínimo 3 chars
        return 0.75 + (shorter / longer) * 0.2
```

**Impacto**: Eliminados falsos positivos en clustering de entidades.

### 3.2 Normalización de Scores Inflada en MorphoCorefMethod (CORREGIDO)

**Archivo**: `nlp/coreference_resolver.py`

**Problema**: `MorphoCorefMethod` usaba `(score + 0.5) / 1.5` para normalizar scores, lo cual:
- Score 0.0 → 0.33 (infla)
- Score 0.5 → 0.67 (infla)
- Score 1.0 → 1.00 (correcto)

`HeuristicsCorefMethod` usaba `min(1.0, max(0.0, score))` — sin inflación.

Los scores de ambos métodos se combinaban en votación, pero con escalas diferentes.

**Fix**: Unificado a `min(1.0, max(0.0, score))` en ambos métodos.

### 3.3 Género NEUTRAL Inflando Scores (CORREGIDO)

**Archivo**: `nlp/coreference_resolver.py`

**Problema**: Cuando anaphor o candidato tenían género NEUTRAL (ej: "su", "le"), se daba +0.2 de bonificación parcial. Esto causaba que pronombres neutrales matchearan con todo el mundo.

Además, las penalizaciones de género eran asimétricas: +0.4 match / -0.3 mismatch.

**Fix**:
- NEUTRAL → 0.0 (sin información, sin efecto)
- Simétrico: +0.35 match / -0.35 mismatch

### 3.4 Excepción Silenciada en `semantic_fusion.py` (CORREGIDO)

**Archivo**: `entities/semantic_fusion.py` → `_get_fusion_threshold()`

**Problema**: `except Exception: pass` sin logging. Si fallaba obtener el threshold, el error desaparecía silenciosamente.

**Fix**: Añadido `logger.debug()` con información del error.

## 5. Problemas Técnicos Pendientes

### 5.1 Overfitting y Generalización

| Área | Problema | Severidad | Recomendación |
|------|----------|-----------|---------------|
| Hipocorísticos | Diccionario solo cubre España (no Latam: Güicho, Chente) | BAJA | Ampliar con variantes regionales |
| Gender inference | Gazetteer de ~240 nombres, sesgo hacia español peninsular | BAJA | Ampliar con nombres latinos/árabes comunes |
| Grammar rules | `FEMININE_WITH_EL` tiene 24 sustantivos hardcoded | MEDIA | Evaluar si faltan más (el hambre, el hada, etc.) |
| Clustering threshold | `CLUSTERING_SIMILARITY_THRESHOLD = 0.45` hardcoded | BAJA | Considerar auto-tuning o parametrizable por proyecto |
| Coref weights | Pesos de votación (30/35/20/15) hardcoded | MEDIA | Sin evidencia empírica que los sustente |

### 5.2 Arquitectura — Archivos Grandes

| Archivo | LOC | Recomendación |
|---------|-----|---------------|
| `coreference_resolver.py` | 1946 | Extraer cada método a su propio módulo (`coref_embeddings.py`, `coref_heuristics.py`, etc.) |
| `scope_resolver.py` | 1710 | Separar scoring de resolución |
| `semantic_fusion.py` | 1644 | Extraer diccionario a archivo separado, separar lógica de merge |
| `unified_analysis.py` | ~1200 | Separar orquestación de lógica de análisis |

### 5.3 Patrones de Código Problemáticos

1. **Silent exception swallowing**: Quedan ~5 instancias de `except Exception: pass` sin logging en el codebase. Cada una debería al menos loggear el error.

2. **Thresholds mágicos**: ~20 constantes numéricas hardcoded sin documentación empírica:
   - `CLUSTERING_SIMILARITY_THRESHOLD = 0.45`
   - `MIN_FUSION_THRESHOLD = 0.52`
   - Bonos de score: +0.6 hipocorístico, +0.35 género, -0.35 género mismatch
   - Sin benchmark que valide estos valores

3. **Import circular prevention**: Varios módulos usan lazy imports dentro de funciones para evitar ciclos. Esto funciona pero dificulta el análisis estático.

### 5.4 `split_sentences()` — Consolidación Pendiente

Se identificaron 7+ implementaciones de partición de texto en oraciones:
- `nlp/sentence_utils.py` (la canónica)
- `corrections/detectors/clarity.py`
